This schema provides a backend foundation for TaskFlow Pro task manager. The main table `app_e590_tasks` stores tasks with title, priority (high/medium/low), and completion status, all scoped to individual users via RLS policies. The schema includes realtime publication for potential live updates and a storage bucket for file attachments. An edge function `sync-tasks` is provided to migrate localStorage tasks to Supabase when users authenticate, enabling a smooth transition from local-only to cloud-backed persistence. All tables are prefixed with `app_e590_` for isolation within the shared Supabase project.
